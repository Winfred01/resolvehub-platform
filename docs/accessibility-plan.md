# Accessibility Plan

## Target

Aim for WCAG 2.1 AA basics for the MVP demo.

## Requirements

- Semantic headings and landmarks.
- Form labels and descriptions.
- Keyboard navigation for ticket lists, filters, modals, and Kanban.
- Visible focus states.
- Color contrast checks.
- Error messages tied to fields.
- Screen-reader friendly status updates.
- Responsive layouts without content overlap.

## Testing

- Component-level accessibility tests.
- Keyboard-only manual smoke tests.
- Automated scans for key pages.
- Review of dashboard charts for textual equivalents.

## Risks

Kanban drag-and-drop can be inaccessible if keyboard alternatives are missing. The MVP must include buttons or menus for status transitions.
