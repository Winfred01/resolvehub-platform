# Security Policy

## Supported Versions

ResolveHub is currently in planning and repository foundation. No production version is supported yet.

## Vulnerability Reporting

Use GitHub Security Advisories or the repository's private security reporting flow when available. Do not disclose secrets, exploit details, or private user data in public issues.

## Secret Rotation

If a secret is ever committed:

1. Stop all deployment or push activity.
2. Rotate the secret at the provider.
3. Audit commit history.
4. Replace local credentials.
5. Document the incident without copying the secret value.

## Demo Data Limits

Demo data must be fictional. Demo accounts must not access real data. Demo credentials should be generated by the deployment environment or a safe public demo mechanism, not hardcoded fixed personal passwords.

## Dependency Updates

Future implementation work should add dependency scanning for frontend, backend, analytics, and containers.

## Security Response

Security fixes should be prioritized, tested, reviewed, and released with clear notes about user impact and required rotation steps.
